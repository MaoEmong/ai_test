# TASK06: 테스트 결과 기록

## 테스트 환경
- 서버: uvicorn `app.main:app`
- 엔드포인트: `POST /run`

## 테스트 1: 데이터 전체조회
요청:
```
{"text":"전체 상품 조회","confirm":false}
```
응답(message):
- "총 3건을 찾았어요. 예: (id=1, name=사과, price=1000, qty=5), (id=2, name=바나나, price=500, qty=10), (id=3, name=딸기, price=800, qty=2)"

결과:
- 정상 응답
- 자연어 메시지 출력 확인

## 테스트 2: '사과' 데이터 조회
요청:
```
{"text":"사과 데이터 조회","confirm":false}
```
응답(message):
- "총 3건을 찾았어요. 예: (id=1, name=사과, price=1000, qty=5), (id=2, name=바나나, price=500, qty=10), (id=3, name=딸기, price=800, qty=2)"

결과:
- 정상 응답
- 필터 미적용(전체 조회로 처리됨)

## 원인 분석
- LLM 계획 생성에서 `where`가 비어 있어 필터가 적용되지 않음.
- 사용자 입력에 특정 키워드가 있어도 조건 추출을 못하는 경우가 있음.

## 후속 개선안
- LLM 프롬프트에 한국어 예시/규칙 추가 ("사과" -> `name LIKE '%사과%'`)
- 또는 클라이언트 입력 규칙 강화 ("name=사과" 형태 유도)
